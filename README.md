ОБРАЗОВАТЕЛЬНАЯ ПЛАТФОРМА (JDBC-версия)

Проект на Java 21 + JavaFX для управления курсами, студентами и сертификатами.
Взаимодействие с базой данных PostgreSQL через низкоуровневый JDBC и пул соединений HikariCP.

АРХИТЕКТУРА
Приложение построено по трёхслойной модели: model (сущности), dao (доступ к данным), service (бизнес-логика). Графический интерфейс реализован в контроллерах с FXML. Настройки подключения загружаются из application.properties. Пакет db содержит утилиты для работы с БД.

ПАКЕТ db
ConnectionManager — синглтон, управляющий пулом соединений HikariCP. При первом вызове загружает конфигурацию из application.properties (db.url, db.username, db.password, размер пула, таймауты) и создаёт HikariDataSource. Предоставляет статический метод getConnection() для получения соединения, которое после использования возвращается в пул при закрытии. Статический метод close() корректно завершает работу пула при остановке приложения. Приватный конструктор запрещает создание экземпляров. Ленивая инициализация с двойной проверкой гарантирует потокобезопасность.

SchemaInitializer — инициализирует схему базы данных и заполняет тестовые данные. Вызывается однократно при старте приложения. Метод initialize() сначала выполняет SQL-файл schema.sql, содержащий команды CREATE TABLE для всех 10 таблиц (users, teacher, student, course, module, lesson, test, course_progress, lesson_progress, sert). Затем в отдельном транзакционном блоке seedTestData() добавляет начальные записи: 10 пользователей (из них 3 преподавателя, 7 студентов), курсы, модули, уроки, тесты, прогресс и один сертификат. Если данные уже есть, заполнение пропускается.

ПАКЕТ model
Содержит простые POJO-классы, отражающие таблицы базы данных. Каждый класс имеет приватные поля, конструкторы, геттеры и сеттеры. Связи между таблицами реализованы через числовые внешние ключи (например, Course хранит teacherId, а не объект Teacher).
- User (id, name, email, passwordHash, registrationDate)
- Teacher (teacherId, userId, name, email, passwordHash, registrationDate) — расширяет пользователя ролью преподавателя
- Student (studentId, userId, zachId, name, email, passwordHash, registrationDate) — добавляет номер зачётной книжки
- Course (courseId, teacherId, name, description, duration)
- CourseModule (moduleId, courseId, name, description)
- Lesson (lessonId, moduleId, name, description)
- Test (testId, lessonId, name)
- CourseProgress (progressId, courseId, userId, progress) — прогресс студента по курсу (not_started, in_progress, completed)
- LessonProgress (progressId, courseProgressId, lessonId, userId, progress) — прогресс по отдельному уроку
- Sert (sertId, courseId, userId) — выданный сертификат

ПАКЕТ dao
Каждый DAO-класс отвечает за CRUD-операции с соответствующей таблицей. Шаблон построения: метод получает соединение через ConnectionManager, выполняет SQL-запрос (через Statement или PreparedStatement), обрабатывает ResultSet и преобразует строки в объекты модели с помощью приватного метода mapRow(). Использование try-with-resources гарантирует закрытие ресурсов.

Типичный DAO включает:
- findAll() — SELECT всех записей с сортировкой по ID
- findById(int id) — поиск по первичному ключу, возвращает Optional
- insert(T obj) — вставка новой записи, возвращает сгенерированный ID и устанавливает его объекту
- update(T obj) — обновление существующей записи по ID
- delete(int id) — удаление по ID, возвращает boolean (успех операции)

Некоторые DAO имеют дополнительные методы выборки по внешним ключам (findByCourse, findByUser, findByModule и т.п.). Все запросы параметризованы для защиты от SQL-инъекций.

SertDao дополнительно содержит метод issueCertificateIfCompleted(int userId, int courseId), реализующий транзакционную логику выдачи сертификата. Соединение переводится в ручной режим (setAutoCommit(false)), проверяется статус прохождения курса (progress = 'completed'), и если курс завершён и сертификат ещё не выдан — создаётся запись в sert и транзакция фиксируется. В противном случае происходит откат и выбрасывается исключение.

ПАКЕТ service
CrudDemoService демонстрирует все CRUD-операции. Использует экземпляры всех DAO. Методы:
- demoCreate() — последовательно создаёт пользователя, преподавателя, курс, модуль, урок и тест. Каждый следующий объект использует ID, полученный при сохранении предыдущего.
- demoRead() — выводит в консоль список всех курсов и пользователей.
- demoUpdate() — находит пользователя с ID=1 и изменяет его email на "updated_старый_email".
- demoDelete() — создаёт временного пользователя, выводит его ID, затем удаляет и проверяет удаление несуществующего ID.
- demoTransaction() — вызывает метод выдачи сертификата для пользователя 4 и курса 1 (ожидается ошибка, так как курс не завершён).

BusinessQueryService содержит 8 методов для аналитических отчётов, каждый из которых выполняет SQL-запрос и возвращает ObservableList<Map<String, Object>> для отображения в JavaFX-таблицах:
- getStudentCountByCourseData() — количество уникальных студентов на каждом курсе (LEFT JOIN, GROUP BY)
- getCourseProgressDetailsData(int courseId) — имена студентов и их статус по указанному курсу
- getTopTeachersData() — топ-3 преподавателя по количеству студентов, записанных на их курсы
- getIssuedCertificatesData() — список выданных сертификатов с именами студентов и названиями курсов
- getUserActivityData() — для каждого пользователя: сколько курсов начато и сколько завершено
- getCourseTeacherListData() — список всех курсов с указанием преподавателей
- getCourseModuleLessonListData() — иерархия курсов, модулей и уроков
- getStudentCourseTeacherProgressData() — сводный отчёт по студентам, их курсам, преподавателям и статусу прогресса

Все методы используют PreparedStatement для параметризованных запросов. Результаты формируются в виде списка карт (Map), где ключи — названия столбцов, значения — объекты (String или Integer). Сервис не зависит от графического интерфейса, что позволяет легко тестировать его отдельно.

ПАКЕТ controllers
MainController — основной контроллер JavaFX, управляющий двумя режимами: "CRUD Demo" и "Business Queries". При инициализации создаются панели для каждого режима. CRUD-панель содержит вертикальное меню с кнопками операций и область для отображения SQL-запроса и результата. Бизнес-панель содержит меню с кнопками отчётов и динамическую таблицу с фильтрами.

Режим CRUD Demo: при нажатии кнопки вызывается соответствующий метод CrudDemoService, а результат (вывод в консоль) перехватывается через перенаправление System.out в ByteArrayOutputStream и отображается в текстовом поле. Также показывается SQL-запрос, связанный с операцией.

Режим Business Queries: для каждого отчёта создаются столбцы таблицы и устанавливаются обработчики фильтров (текстовое поле, спиннер). Фильтрация реализована через FilteredList. Кнопка "Обновить все данные" перезагружает исходные списки из сервиса.

ТОЧКА ВХОДА
Класс app.MainApp расширяет Application. При старте вызывает SchemaInitializer.initialize() для создания таблиц и заполнения данными (если БД пуста). Затем загружает FXML-разметку main.fxml, создаёт сцену 1200x700 с подключением CSS-стилей и отображает главное окно с заголовком "Образовательная платформа". При закрытии вызывает ConnectionManager.close() для корректного завершения пула соединений.

БЫСТРЫЙ СТАРТ
1. Создайте базу данных PostgreSQL.
2. Настройте src/main/resources/application.properties:
   db.url=jdbc:postgresql://localhost:5432/имя_бд
   db.username=пользователь
   db.password=пароль
3. Соберите и запустите приложение: mvn clean javafx:run
4. Таблицы и тестовые данные создадутся автоматически при первом запуске.
